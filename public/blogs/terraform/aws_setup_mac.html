まるちゃん 2020-08-24 setup

# 1. IAM の作成

  1. AWSコンソールアクセスして､IAMページに遷移
  2. 【ユーザー】を選択して､【ユーザーを作成】を選択
    • step1. ユーザー編
      • ユーザー名 : 任意の名前を入力
      • プログラムによるアクセス : 選択
    • step2. ポリシー編
      • 【既存のポリシーを直接アタッチ】を選択
      • 状況によって与えるポリシーを変える
        • ポリシー名 : AdministratorAccess
    • step3. タグ編(【タグの追加(オプション)】項目については､任意)
      • 項目 : Terraform
      • 値 : Study
    • step4. 確認編

ローカルで AWS CLI 2 のセットアップページ

  • AWS CLI バージョン 2 セットアップ

# 2. 【direnv】セットアップ方法

direnvとは?!

ディレクトリ毎に環境変数を定義して､そのディレクトリがカレントになった時だっけ環境変数を有効/無効にしてくれる素晴らしいツール

GitHub direnv

# 2.1. 【direnv】インストール

$ brew install direnv
1

# 2.2. 【direnv】シェルのフック設定

# 2.2.1. zsh の場合

  • 下記の様に追記
$ vim ~/.zshrc

eval "$(direnv hook zsh)"
1
2
3
  • 反映
source ~/.zshrc
1

# 2.2.2. bash の場合

  • 下記の様に追記
$ vim ~/.bashrc

eval "$(direnv hook bash)"
1
2
3
  • 反映
source ~/.bashrc
1

# 2.3. AWS プロファイル登録

  • どのエディターで起動するかを指定
    • vim の所は任意
$ export EDITOR=vim
1
  • direnv edit . を実行すると指定したエディターが立ち上がり､下記を追記
    • 自分のアクセス･シークレットキーを入力後に､実行させたいリージョンを指定
$ direnv edit .

export AWS_ACCESS_KEY_ID="自分のアクセスキーを入力"
export AWS_SECRET_ACCESS_KEY="自分のシークレットキーを入力"
export AWS_REGION="リージョンの選択"

# 3. Terraform の文法解説

Terraform はHCLというHashicorm遠くじの言語を使用している｡また､HCLはJsonとの互換性がある｡

# 3.1. 話し言葉で理解

  • resource : 作業したいモノを取り扱うことを示します
  • data : 情報を定義
やりたいこと "どの機能を利用するか" "このリソースの名称" {
    設定項目 = "設定内容"
}
1
2
3

# 3.2. 公式の言語構成紹介

resource "aws_vpc" "main" {
    cidr_block = var.base_cidr_block
}

<BLOCK TYPE> "<BLOCK LABEL" "<BLOCK LABEL>" {
    # Block body
    <IDENTIFIER> = <EXPRESSION> # Argument
}
